# Лабораторна робота №6. Основи роботи з віртуальними машинами

**Тривалість**: 4 акад. години.

## Порядок виконання роботи 

У наступних пунктах робіть копії екранів після виконання пунктів для звітів.

### 1. Встановлення ВМ з ОС Windows

Якщо у Вас ПК має ресурси принаймні в обсязі CPU Intel/AMD 2 ГГц / RAM 16 ГБ / Диск 20 ГБ (вільних), то виконуйте наступний пункт по варіанту 1, якщо ні - по варіанту 2.

#### Варіант 1: виконання для ПК з достатніми ресурсами

- [ ] Виконайте [Встановлення та налаштування ВМ з Windows 10](https://github.com/asu-in-ua/atpv/blob/main/vm/vbox/labwin10.md)

#### Варіант 2: виконання для ПК з обмеженими ресурсами

- [ ] Виконайте [Встановлення та налаштування ВМ з Windows XP](https://github.com/asu-in-ua/atpv/blob/main/vm/vbox/lab.md)

### 2. Встановлення ВМ з ОС Debian

- [ ] Виконайте [Встановлення віртуальної машини з ОС Debian: практичне заняття](https://github.com/asu-in-ua/atpv/blob/main/vm/vbox/labdebian.md)

### 3. Формування звіту

У цьому пункті необхідно в робочій папці, яка була створена на першій лабораторній роботі розмістити всі звіти щодо виконання цієї лабораторної роботи.  

- [ ] У робочій директорії для звітів, яка була створена на першій лабораторній роботі:
  - створіть папку `lab6` 
  - у папці `lab6` створіть папку `media` 
  - у папці `lab6` створіть файл `README.md` в якому оформіть звіт по даній лабораторній роботі, у звіті мають бути:
    - зображення копій екранів, розміщення яких зробити в папці `media`; зверніть увагу, що в `README.md` посилання на зображення мають мати відносні шляхи
    - посилання на файли відео, які варто розміщати на YouTube або Google Drive або інших доступних ресурсах (**в саму папку лабораторної роботи відео та інший бінарний контент розміщувати не рекомендується, так як це стане частиною репозиторію Git та GitHub**).
- [ ] У корені робочої директорії змініть файл `README.md` таким чином, щоб пункт `Лабораторна робота 6` посилався на файл `lab6/README.md`
- [ ] Зробіть коміт змін і Push у власний репозиторій для лабораторних робіт GitHub
- [ ] Переконайтеся що все в GitHub відображається коректно
